## 2. `visual_agent.generate_all` 与图片生成接线状态

### 2.1 当前 `generate_all` 行为

文件：`app/backend/app/services/visual_agent.py`

核验范围：`generate_all` 当前位于约 155-248 行。

当前流程：

```text
generate_all
  -> generate_main_image(brief)
  -> generate_white_bg(brief)
  -> generate_scene_images(brief)
  -> generate_selling_points(brief)
  -> generate_video_scripts(brief)
  -> generate_ad_material(brief)
  -> layout_agent.generate_layout(...)
  -> VisualAssetPlanOut(...)
  -> save_asset_plan(...)
  -> return result
```

结论：`generate_all` 当前没有调用 `image_generation_service.generate()`，也没有调用同文件已有的 `generate_images_from_plan()`。它仍然只生成图片/场景图的计划对象和 prompt 文本。

### 2.2 半成品图片渲染辅助方法已存在

同文件约 249-301 行存在：

```text
async def generate_images_from_plan(plan, provider="dalle", width=1024, height=1024) -> dict
```

该方法内部已导入：

```text
from app.models.image_generation_model import ImageGenerationRequest
from app.services.image_generation_service import image_generation_service
```

并对：

```text
plan.main_image.prompt
plan.scene_images[].prompt
```

调用：

```text
await image_generation_service.generate(req)
```

结论：阶段 1 不应新建渲染器，应优先把已有 `generate_images_from_plan()` 接进主生成流水线，并优先使用 `provider="local"` 去除 key 依赖。

### 2.3 `unified_generation_routes.py` 中的图片生成调用状态

文件：`app/backend/app/api/unified_generation_routes.py`

核验结果：

- 约 175 行：同步生成路径调用 `agent.generate_all(...)`。
- 约 201 行：先 `save_asset_plan(... result.model_dump() ...)`。
- 约 207-217 行：只有当 `generate_images` 为真时，才异步 fire-and-forget 调用：

```text
imgs = await agent.generate_images_from_plan(result, provider="mige")
_image_results[project_id] = {"status": "done", "images": imgs}
```

问题：

1. 默认不是主流水线必经路径。
2. 使用 `provider="mige"`，仍依赖外部 key/可用性，不符合阶段 1 local provider 打通要求。
3. 生成出的 `imgs` 写入内存 `_image_results`，没有合并回 `result.asset_plan`，也没有落入 `VisualAsset` 的 `asset_plan`。
4. 因此 `/canvas-assets` 仍然无法从历史资产中提取到 `url`。

### 2.4 488 项 canvas-assets 的真实代码路径

真实生成/提取路径确认如下：

- 数据来源：`app/backend/app/models/visual_asset.py` 的 `VisualAsset.asset_plan` 历史记录。
- 保存入口：`app/backend/app/db/crud_visual_asset_v2.py::save_asset_plan(...)`。
- 生成调用：`app/backend/app/api/unified_generation_routes.py` 调用 `agent.generate_all(...)` 后保存 `result.model_dump()`。
- 资产库提取：`app/backend/app/api/atelier_canvas_routes.py::get_canvas_assets(...)`，约 241 行开始。

`get_canvas_assets` 当前逻辑：

- `main_image`：只有 `main.get("url")` 才追加 `type="image"`。
- `white_bg`：只有 `white_bg.get("url")` 才追加 `type="image"`。
- `scene_images`：只有 `scene.get("url")` 才追加 `type="image"`。
- `selling_points`：追加 `type="graphic"`，只有文字，无 `url`。
- `video_scripts`：追加 `type="video"`，`preview_url` 当前为空。
- `ad_material`：追加 `type="graphic"`，只有文字，无 `url`。

结论：488 项来自历史 `VisualAsset.asset_plan` 的文字型 `selling_points / video_scripts / ad_material`。由于图片计划没有真实 `url`，图片类资产被过滤掉，最终只有 graphic/video。
